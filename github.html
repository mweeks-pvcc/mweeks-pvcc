GitHub Q&A

  1. Q. What is a version control system?
    2. A. Is the practice of tracking and managing changes to software code
  2. Q. What is Git?
    3. A. It is a free and open source distributed version control system
  3. Q. What is GitHub
    4. A. Is a web-based version control and collaborative platform for software developers
  4. Q. What is the difference between Git and GitHub?
    5. A. Git is a version control system that lets you manage and keep track of your souce code history. GitHub is a cloud-based hosting service that lets you mangage Git repositories.
  5. Q. Who started Github and how was it started? A. It was started by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett, and Scott Chacon. Wanstrath and Preston-Werner needed GitHub for their own work, and so they created it for that need, they discovered that they were forking codes branches ans collaborating on programming projects.

GitHub Terminology

  1. Repository
    2. Definiition: a central location in which data is stored and managed.
  2. Commit
    3. Definiition:is an operation which sends the latest changes of the source code to the repository, making these changes part of the head revision of the repository.
  3. Fork
    4. Definiition:is an operation whereby a process creates a copy of itself.
  4. Push
    5. Definiition:is the delivery of information on the Web that is initiated by the information server rather than by the information user or client , as it usually is.
  5. Pull Requests
    6. Definiition: let you tell others about changes you pushed to a branch in a repository on GitHub.
  6. Workflows
    7. Definiition:a configurable automated process that will run one or more jobs.
  7. Issues
    8. Definiition:let you track your work on GitHub, where development happens.
  8. Raw Button
    9. Definiition:command is used so you can view or copy the raw content of a file without any styling.
  9. Blame Button
    10. Definiition:command is used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was.
